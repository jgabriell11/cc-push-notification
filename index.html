claude-code push v1 2026 5min quick start · 01

push.
_noti_fications.

Quando o Claude Code precisa do seu input, permissão ou decisão, o terminal não precisa gritar. Este guia ativa notificações nativas e push para o seu celular em 5 minutos.

tl;dr
  1. Ative agentPushNotifEnabled em ~/.claude/settings.json
  2. Adicione um hook Notification com curl para o ntfy.sh
  3. Instale o app ntfy no celular e assine o topic
  4. Pronto. Push dispara sempre que o Claude espera por você
01
native · settings.json

Ativar notificação nativa

O Claude Code tem uma flag interna que dispara notificações de sistema (macOS Notifications, Linux D-Bus, Windows Toast) quando o agente aguarda input.

~/.claude/settings.json
{
  "agentPushNotifEnabled": true
}

Não confunde com --notification via flag de CLI. A chave certa é agentPushNotifEnabled. Funciona em qualquer terminal: iTerm, Warp, Ghostty, Terminal.app.

02
remote · ntfy.sh

Push pro celular com ntfy.sh

Notificações de sistema só funcionam no computador. Para alertas no celular enquanto você está longe, use o ntfy.sh — pub-sub HTTP, zero setup, sem cadastro.

Escolha um topic aleatório (ex: claude-soier-x9k2). Adicione o hook Notification ao settings.json:

~/.claude/settings.json
{
  "agentPushNotifEnabled": true,
  "hooks": {
    "Notification": [
      {
        "type": "command",
        "command": "curl -s -d 'Claude Code precisa de você' https://ntfy.sh/claude-soier-x9k2",
        "timeout": 5000
      }
    ]
  }
}

Baixe o app ntfy:

iOS · App Store Android · Play Store

Assine o mesmo topic para receber push de verdade no celular.

Segurança: Tópico = senha. Quem descobrir o nome pode te mandar pushes. Use strings longas e aleatórias.
03
remote · pushover

Alternativa premium: Pushover

Para entrega garantida, alta prioridade, retry automático e sons customizados, use Pushover (pagamento único de $5 por plataforma).

~/.claude/settings.json
{
  "hooks": {
    "Notification": [
      {
        "type": "command",
        "command": "curl -s -X POST https://api.pushover.net/1/messages.json -d 'token=APP_TOKEN&user=USER_KEY&message=Claude Code precisa de você&priority=1'",
        "timeout": 5000
      }
    ]
  }
}

Substitua APP_TOKEN e USER_KEY com suas credenciais do dashboard do Pushover. Priority 1 dispara som mesmo no modo silencioso.

04
macos · osascript

Notificação macOS rica (sem servidor)

Para Mac: banners customizados, som e ícone — sem dependência de serviço externo.

~/.claude/settings.json
{
  "hooks": {
    "Notification": [
      {
        "type": "command",
        "command": "osascript -e 'display notification \"Aguardando input\" with title \"Claude Code\" sound name \"Glass\"'"
      }
    ]
  }
}

Sons disponíveis:

Glass Ping Pop Hero Submarine

Lista completa: /System/Library/Sounds/

05
triggers · when it fires

Quando o hook dispara

O evento Notification roda em situações específicas — não a cada token gerado.

Trigger Quando
waiting-for-input Claude concluiu o turno e aguarda sua resposta
permission-required Tool call bloqueada aguardando aprovação
agent-idle Sub-agente ocioso aguardando comando
task-complete Custom — você dispara via hook Stop

Para notificações no fim de tarefas longas (não só quando aguarda input), adicione um hook Stop com o mesmo comando curl.

06
debug · troubleshooting

Não está tocando?

Sintoma Fix
Hook não dispara Execute /hooks no Claude Code para listar hooks registrados. Se não aparecer, o JSON está malformado
Curl trava sessão Sempre defina "timeout": 5000. Sem timeout, request presa trava o agente
macOS não notifica Ajustes do Sistema → Notificações → Terminal/iTerm/Warp → ative "Permitir Notificações"
ntfy não chega no celular Confirme que o topic no app é exatamente igual ao do comando curl. Case-sensitive
Quero testar sem esperar Execute diretamente: curl -d "test" https://ntfy.sh/seu-topico
